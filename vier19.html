Estructura semantica de un html5

Header:Define el contenido que debe considerarse en la informacion indtroductoria de una pagina o seccion

nav: Son los enlaces del menu de navegacion principal

aside:

Sirve para definir el contenido que es menos importante. a menudo se utiliza para barras laterales e informacion complementaria pero no principal

main:

Es el cuerpo de una pagina.

Debe haber uno solo por pagina

article: define el contenido autonomo que puede ser independiente de la pagina o el sitio en el que se encuentra por ejemplo una entrada a un blog

section : se utiliza para contenido cercano a un tema similar, una etiqueta de section difiere de una etiqueta article pero no es necesariamente independiente

elemento details: se utiliza para crear un widget de revelacion que permite al usuario mostrar u ocultar contenido adicional en una pagina web el contenido dentro de un elemnto details esta oculto de forma predeterminada y solo se mustra cuando el usuario interactua con el

summary:define un encabezado visible para un elemento details
